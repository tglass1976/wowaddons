# World of Warcraft Namespaces Source: https://community.developer.battle.net/documentation/world-of-warcraft/guides/namespaces Namespaces allow JSON documents to be published contextually in relation to a specific patch or point in time, and group documents by category. ## Namespace Categories ### Static Namespaces Contain data directly related to underlying game systems. Expected to change per-patch (items, achievements, etc.). ### Dynamic Namespaces Contain data published in response to in-game events (leaderboards, WoW Token price, auction house). Changes frequently. ### Profile Namespaces Contain data contextual to a specific character or Battle.net account (character achievements, guild roster). - Character data updated on character logout. - Guild data updated at regular intervals. ## Available Namespaces | Category | Pattern | |----------|---------| | Static | static-{region} | | Dynamic | dynamic-{region} | | Profile | profile-{region} | Region identifiers: us (North America), eu (Europe), kr (Korea), tw (Taiwan), cn (China) Examples: static-us, dynamic-cn, profile-tw ## Specifying a Namespace Every WoW API request must include a namespace. | Method | Format | Example | |--------|--------|---------| | Header | Battlenet-Namespace: {namespace} | Battlenet-Namespace: static-us | | Query | ?namespace={namespace} | ?namespace=static-us |
